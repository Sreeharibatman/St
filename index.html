Frequently Asked Questions about GitHub

1. How do I create a new repository?

To create a new repository on GitHub, you can follow these steps:

  1. Log into your Github account
    In the upper-right corner of any page, select +,then click New repository
  2. Optionally, to create a repository with the directory structure and files of an existing repository, select the Choose a template dropdown menu and click a template repository. You'll see template repositories that are owned by you and organizations you're a member of or that you've used before. For more information, see "Creating a repository from a template"

  3. Optionally, if you chose to use a template, to include the directory structure and files from all branches in the template, and not just the default branch,select Include all branches.

  4. Use the Owner dropdown menu to select the account you want to own the repository.

  5. Type a name for your repository, and an optional description.

  6. Choose a repository visibility. For more information, see "About repositories."

  7. If you're not using a template, there are a number of optional items you can pre-populate your repository with. If you're importing an existing repository to GitHub, don't choose any of these options, as you may introduce a merge conflict. You can add or create new files using the user interface or choose to add new files using the command line later. For more information, see "Importing an External Git respository using the command line," "Adding a file to a repository,"and"Addressing merge conflicts"

    • You can create a README, which is a document describing your project. For more information, see "About READMes">

    • You can create a .gitignore file, which is a set of ignore rules. For more information, see "Ignoring files"

    • You can choose to add a software license for your project. For more information,"Licensing a repository "

  8. Optionally, if the personal account or organization in which you're creating uses any GitHub Apps from GitHub Marketplace, select any apps you'd like to use in the repository.

  9. Click Create repository

  10. At the bottom of the resulting Quick Setup page, under "Import code from an old repository", you can choose to import a project to your new repository. To do so, click Import code.

2. Can I contribute to someone else's repository?

Yes, you can contribute to other repositories by

  1. Fork the original repository: Forking will create an exact duplicate of the original repository to your own account.

  2. Clone the forked repository onto your local machine

  3. Create a branch, make changes, commit

  4. Push the new branch to your fork

  5. Now go to the original repository, go to the pull requests tab and open a new pull request. GitHub might even suggest creating a pull request without even needing to go to the pull requests tab.

  6. Explain what the changes are, and why they are necessary

  7. Someone from the original repository will review your code, and might request some changes or they'll reject the PR

  8. Assuming the PR was not rejected, it should be merged by one of the maintainers at some point

3. How can I clone a repository to my local machine?

To clone a repository, use the following command in your terminal

  1. On GitHub.com, navigate to the main page of the repository

  2. Above the list of files, click < > Code.

  3. Copy the URL for the repository.

  4. Open terminal

  5. Change the current working directory to the location where you want the cloned directory

  6. Type git clone, and then paste the URL you copied earlier.

  7. Press Enter to create your local clone.

4. What is a pull request?

A Pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub. Once a pull request is opened, you can discuss and review the potential changes with collaborators and add follow-up commits before your changes are merged into the base branch.
